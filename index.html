再看PNG

首先是关于透明png在ie6的显示

相比滤镜这次用vml来代替，实现方式也更优雅。缺陷是额外引入了DD_belatedPNG.js 这个js文件 不过也就几K大小。

使用方式如下，可以看到只要引入指定js文件后用DD_belatedPNG.fix(‘.png_bg’) 指定css选择器即可

可以用在使用了透明png的img标签上，也可以用在使用了透明png做背景的标签上

[js]
<img src="some.png" class="png_bg" />

<!–[if IE 6]>
<script src="DD_belatedPNG.js"></script>
<script>
/ EXAMPLE /
DD_belatedPNG.fix(‘.png_bg’);

/ string argument can be any CSS selector /
/ .png_bg example is unnecessary /
/ change it to what suits you! /
</script>
<![endif]–>
[/js]

不过作者在11年已经放弃维护了，毕竟ie6在外国早就淘汰了嘛

这里是官网

http://www.dillerdesign.com/experiment/DD_belatedPNG/

第二是关于png压缩的网站。

做手机端一些css动画时常切出png24（即fireworks中png32）的透明图片。这些图片通常都不可接受的大，动辄几百K。

这里是立华推荐的压图网站，tinypng.com

“有了这个我们才敢这么做动画啊”立华语。

200多K到20多K的感觉真是爽啊。

关于rem

em是相对于当前元素或继承自祖先元素的文字大小作基准
rem相对于页面根节点设置的文字大小作基准
通常设置html{font-size:62.5%},则页面中元素要设置12px大小的样式可以写成{font-size:1.2rem;}方便计算

mac命令行化操作体验之svn常用命令（持续更新）

所有的网上都能查到。记忆无用，多用自然记得。
以下是自己常用的svn 命令记录:
svn —version #查看当前版本
svn up #更新服务器版本到本地
svn ci -m “说明文字” . #提交本地版本到服务器；注意最后的点是路径，当前目录

关于设备分辨率

关于设备分辨率

1. 分辨率分为物理分辨率和逻辑分辨率

2. 物理分辨率是硬件显示的分辨率（不可变）

3. 逻辑分辨率是软件（系统）用来解析的分辨率（可变）

4. 不同分辨率下图像大小发生变化，指的是逻辑分辨率

5. 非retina iPad（iPad2及以下）物理分辨率1024x768 逻辑分辨率 1024x768

6. retina iPad（new iPad及以上）物理分辨率2048×1536 逻辑分辨率 1024x768

前端方面

1. css中的像素指的是逻辑像素

2. 适配某个设备的分辨率指的是逻辑分辨率

3. css3 media query 检测的是设备的逻辑分辨率

4. -webkit-device-pixel-ratio检测的是物理分辨率与逻辑分辨率的比值（物理分辨率中的长除以逻辑分辨率的长，或者是两个宽相除）；对应的js属性是window.devicePixelRatio；这个值retina屏苹果产品是2，非retina屏是1，安卓中有1.5的情况

  • 同样15’的mpb换了屏幕的那台比起普通的15‘mbp看网页宽度变小两边空白变多和文字变小，是因为它在增大了物理分辨率的同时支持了1680x1050的逻辑分辨率；

  • 13’rmbp文字、视频显示效果比普通13’mbp清晰得很多但是没有网页、文字大小改变，是因为只有物理分辨率变大了而逻辑分辨率没有变；

为什么我是一个德国队球迷

06年世界杯对我来说是个转折年。那之前我痴迷齐达内的大师风范，便也爱屋及乌的祈祷法国队取胜。但其实我是不怎么喜欢法国队本身的，之所以支持，也只是为了看到足球生涯行将结束的齐达内可能成为绝唱的每一场比赛。

而对德国队的感情则完全不同。

06年以前的德国队我并不不认同，对它的认知只限于巴拉克和卡恩，还有更早的埃芬博格，以及98年廉颇老矣的克林斯曼。那时对德国队的印象正是98年老迈的德国队留下的：高大壮，擅长头球和身体对抗，长传冲吊，毫无技术含量。

直到06年世界杯，以主教练身份回归的克林斯曼从根本上改变了这只球队的风格：严谨，稳定，讲究整体配合；还有这只球队的技战术特点：强调脚下的传控，立体式多元化的进攻方式，队员间组成的小组战队式的局部与浑然而成的整体。

他们进攻起来通过长短结合的传递，行云流水，像潮水一样推进极具观赏性；他们后卫和门将稳定的发挥又确保整支球队无所顾忌的进攻。

我不太喜欢拉姆，不太喜欢厄齐尔，不太喜欢穆勒，不太喜欢施魏因斯泰格……这几年的德国队没有一个球员是我喜欢的。

但是他们组成的整体，踢得让我赏心悦目。

这就是为什么我是一个德国队球迷。

写在巴西世界杯1/4决赛vs法国，大战前

apply和call

apply和call的相同点：改变作用域，this的指向；
不同点，传递的参数不同

[js]
function b(){
a.apply(this,arguments) //a.apply(this,["1","2"…])
//a.call(this,"1","2"…)
}
[/js]

apply和call使用的例子

[js]
function Cc(message){
this.message = message;
}
Cc.prototype = {
constructor:Cc,
showMessage:function(msg){
var msg = msg ||this.message
console.log(msg);
return msg
}
}
var ac = new Cc("hah");

function Bb(message){
    this.message = message;
}
Bb.prototype = {
    constructor:Bb,
    showing:function(){
        console.log(this)
        return ac.showMessage.apply(this,[&quot;fk&quot;])
    }
}
var bc = new Bb(&quot;who&quot;);
bc.showing();

[/js]

each函数的实现

[js]
function each(tagname,callback){
var lists = document.getElementsByTagName(tagname);
for(var i=0;i<list.length;i++){
callback.call(lists[i]);
}

}
each(&quot;li&quot;,function(){
       console.log(this);
})

[/js]

另一个例子

[js]
function Person(name,age,job){
this.name = name;
this.age = age;
this.job = job;
this.sayName = function(){
alert(this.name);
}
}
var jack = new Person("jack","18","f2e")
var obj = {
name:"won",
who:function(){
jack.sayName.apply(this);
}
}
obj.who()
[/js]

关于事件捕获和冒泡

关于事件捕获和冒泡，讲得很清楚了。
http://www.liloy.info/archives/103.html

addEventListener 的第三个参数，true即为捕获阶段监听，false则是冒泡阶段监听。
都监听需要绑定两次，先true再false

attachEvent 先绑定后执行 （ie9之前ie专用）

兼容代码

[js]
if (el.addEventListener) {
el.addEventListener(‘click’, modifyText, false);
} else if (el.attachEvent) {
el.attachEvent(‘onclick’, modifyText);
}
[/js]
