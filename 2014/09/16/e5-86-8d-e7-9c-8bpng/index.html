再看PNG

首先是关于透明png在ie6的显示

相比滤镜这次用vml来代替，实现方式也更优雅。缺陷是额外引入了DD_belatedPNG.js 这个js文件 不过也就几K大小。

使用方式如下，可以看到只要引入指定js文件后用DD_belatedPNG.fix(‘.png_bg’) 指定css选择器即可

可以用在使用了透明png的img标签上，也可以用在使用了透明png做背景的标签上

[js]
<img src="some.png" class="png_bg" />

<!–[if IE 6]>
<script src="DD_belatedPNG.js"></script>
<script>
/ EXAMPLE /
DD_belatedPNG.fix(‘.png_bg’);

/ string argument can be any CSS selector /
/ .png_bg example is unnecessary /
/ change it to what suits you! /
</script>
<![endif]–>
[/js]

不过作者在11年已经放弃维护了，毕竟ie6在外国早就淘汰了嘛

这里是官网

http://www.dillerdesign.com/experiment/DD_belatedPNG/

第二是关于png压缩的网站。

做手机端一些css动画时常切出png24（即fireworks中png32）的透明图片。这些图片通常都不可接受的大，动辄几百K。

这里是立华推荐的压图网站，tinypng.com

“有了这个我们才敢这么做动画啊”立华语。

200多K到20多K的感觉真是爽啊。

 

15年3.13更新

由于tinypng被组内频繁使用外网ip被封，现在用了Pngyu。

Pngyu，客户端，多平台，本地使用更方遍。对比tinypng压缩比可能差一点，不过影响不大。
